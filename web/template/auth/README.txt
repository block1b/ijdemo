用例：用户登陆
1.目标
    本用例用于验证用户身份。
2.事件流
    2.1基本事件流
    当用户要使用登陆页面时，本用例开始执行.
    1）系统请求用户输入账号，密码，角色。
    2）系统从数据库验证是否为系统用户。
    3）验证通过后，将用户信息保存在session中，跳转至用户角色对应的页面。
    2.2可选流程
    1）账号或密码错误。
    在主流程中若用户输入的账号，密码，角色，与数据库中不一致，在页面提示账号或密码错误，要求重新输入。

3.	特殊需求
4.	前提条件
5.	后置条件
